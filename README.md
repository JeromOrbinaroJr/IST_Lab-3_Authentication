# Виртуальный деканат — модуль аутентификации (Python + SQLite)

Этот репозиторий — моя часть командного проекта: **аутентификация пользователей** для ИС «Виртуальный деканат».
Стек по договорённости команды: **Python + SQLite** (+ простой веб-интерфейс).

## 1) Идея и выбранный вариант

Выбран вариант **«постоянный пароль у каждого пользователя»**:
- пользователь заводится в системе (деканатом/админом) и получает логин + постоянный пароль;
- вход выполняется по `логин + пароль`;
- в БД хранится **не пароль**, а **хэш пароля** (см. поле `password_hash`).

В файле командного проекта по БД встречаются поля вида **«Логин пользователя в системе»** и **«хэш_пароля»** — под это и сделана таблица пользователей.

## 2) Что реализовано

- Веб-сайт «болванка» с формами входа и простыми страницами:
  - `GET/POST /login` — вход по логину/паролю
  - `POST /logout` — выход
  - `GET /` — главная (после входа показывает профиль)
  - `GET /admin/users` — список пользователей (только для роли `admin`)
  - `GET/POST /me/password` — смена пароля (опционально, полезно для демонстрации)
- Хранение пользователей в SQLite:
  - таблица `users` (логин, ФИО, роль, активность, хэш пароля, даты)
- Сессии:
  - вход создаёт серверную сессию через cookie Flask (подписанная cookie с `session`).
- Безопасность (база):
  - хэширование паролей `pbkdf2:sha256` (Werkzeug)
  - проверка роли/доступа на защищённых страницах
  - защита от простых ошибок: неверный логин/пароль, отключённый пользователь

## 3) Быстрый старт (Windows / PowerShell)

### 3.1 Установка зависимостей

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 3.2 Инициализация БД и тестовые пользователи

```powershell
python scripts\init_db.py
python scripts\seed_db.py
```

По умолчанию создаются пользователи для демонстрации:
- `admin / admin123`
- `student1 / student123` (роль `student`)
- `teacher1 / teacher123` (роль `teacher`)
- `dean1 / dean123` (роль `dean`)

База создаётся в `instance\app.db`.

### 3.3 Запуск

```powershell
python app.py
```

Открыть в браузере: `http://127.0.0.1:5000`

## 4) Структура проекта

- `app.py` — точка входа (Flask)
- `app/` — приложение (аутентификация, доступ, работа с БД)
- `app/schema.sql` — схема таблиц для модуля аутентификации
- `templates/` — HTML-шаблоны
- `static/` — стили
- `scripts/` — инициализация и наполнение БД
- `instance/app.db` — SQLite БД (генерируется)

## 4.1 Схема БД (аутентификация)

Основная таблица — `users`:
- `username` — логин (unique)
- `password_hash` — хэш пароля (не хранить пароль в чистом виде)
- `role` — роль (`admin | dean | teacher | student`)
- `is_active` — включён/отключён

Схема находится в `app/schema.sql`.

## 5) Коротко для отчёта (моя часть)

**Цель:** обеспечить вход пользователей в систему «Виртуальный деканат» по постоянному паролю, разграничить доступ к функциям по ролям.

**Роли (минимум для демонстрации):**
- `admin` — администрирование пользователей (просмотр списка)
- `dean` — сотрудник деканата (пример роли с расширенными правами)
- `teacher` — преподаватель
- `student` — студент

**Хранение паролей:** только в виде хэша `password_hash`. При входе пароль пользователя сравнивается с хэшем (`check_password_hash`).

**Разграничение доступа:** реализовано декораторами `login_required` и `roles_required(...)` на маршрутах.

## 6) Примечания

- Для реального использования нужно заменить `SECRET_KEY` в `app/__init__.py` на уникальное значение (сейчас стоит учебное `dev-secret-key-change-me`).
